# Paperless-NGX — docker-compose

Document management system. Scans, OCRs, and indexes documents. Runs on the Debian VM at `192.168.178.141:8010`.

**Host:** Debian VM `192.168.178.141`
**Compose directory:** `/opt/stacks/paperless/`
**UI:** `http://192.168.178.141:8010`

---

## Stack Overview

| Container | Image | Role |
|---|---|---|
| `paperless-webserver-1` | `ghcr.io/paperless-ngx/paperless-ngx` | Main application |
| `paperless-db-1` | `postgres:18` | Database — user `paperless`, db `paperless` |
| `paperless-broker-1` | `redis:8` | Task queue broker |

Container names are auto-generated by Docker Compose from the directory name (`paperless`) and service names (`db`, `broker`, `webserver`). No explicit `container_name:` is set — do not rename the compose directory or the container names will change, breaking `dump-databases.sh`.

---

## Environment Variables

The compose file references `stack.env` for the webserver service configuration. Create this file in the compose directory alongside `docker-compose.yml`.

> ⚠️ The `stack.env` file is not committed to the repo — it must exist on the server. See the [Paperless-NGX documentation](https://docs.paperless-ngx.com/configuration/) for all available settings.

Minimum required content:

```env
PAPERLESS_SECRET_KEY=CHANGE_ME_LONG_RANDOM_STRING
PAPERLESS_TIME_ZONE=Europe/Berlin
PAPERLESS_OCR_LANGUAGE=deu+eng
```

## Postgres Credentials

The postgres credentials are hardcoded in the compose file for simplicity. For a homelab this is acceptable. The database is not exposed outside the Docker network.

```
User:     paperless
Password: paperless
Database: paperless
```

> ⚠️ **Bug in `dump-databases.sh`:** The dump script uses `-U postgres` for all containers, but `paperless-db-1` only has the `paperless` user. Dumps for Paperless will fail until this is fixed. See `known-issues.md`.

## Volumes

| Named volume | Purpose |
|---|---|
| `data` | Paperless application data |
| `media` | Document files (PDFs, images) |
| `pgdata` | Postgres data directory |
| `redisdata` | Redis persistence |
| `./export` | Export directory (host bind mount) |
| `./consume` | Consume/inbox directory (host bind mount) |

---

## Backup Coverage

| What | How |
|---|---|
| Database | `dump-databases.sh` → `pg_dump` nightly → TrueNAS |
| Redis broker | `dump-databases.sh` → RDB snapshot nightly → TrueNAS |
| Media files | Included in PBS VM backup → TrueNAS → Filen Cloud |

## docker-compose.yml

```yaml
services:
  broker:
    image: docker.io/library/redis:8
    restart: unless-stopped
    volumes:
      - redisdata:/data

  db:
    image: docker.io/library/postgres:18
    restart: unless-stopped
    volumes:
      - pgdata:/var/lib/postgresql
    environment:
      POSTGRES_DB: paperless
      POSTGRES_USER: paperless
      POSTGRES_PASSWORD: paperless

  webserver:
    image: ghcr.io/paperless-ngx/paperless-ngx:latest
    restart: unless-stopped
    depends_on:
      - db
      - broker
    ports:
      - "8010:8000"
    volumes:
      - data:/usr/src/paperless/data
      - media:/usr/src/paperless/media
      - ./export:/usr/src/paperless/export
      - ./consume:/usr/src/paperless/consume
    environment:
      PAPERLESS_REDIS: redis://broker:6379
      PAPERLESS_DBHOST: db
    env_file:
      - stack.env
    labels:
      - "diun.enable=true"
      - "homepage.group=Documents"
      - "homepage.id=paperless"
      - "homepage.name=Paperless"
      - "homepage.icon=paperless"
      - "homepage.href=http://192.168.178.141:8010"
      - "homepage.description=Document Management"

volumes:
  data:
  media:
  pgdata:
  redisdata:
```
